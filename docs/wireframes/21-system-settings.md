## Interactions

### 1. Update Settings
1. User modifies settings fields
2. Changes are highlighted
3. "Save All Changes" button enabled
4. User clicks save
5. System validates settings
6. Shows success confirmation

### 2. Test Email Configuration
1. User clicks "Test Email Configuration"
2. Opens test modal
3. Enters recipient email
4. Clicks "Send Test Email"
5. System sends email
6. Shows success/failure message

### 3. Configure Integration
1. User clicks "Configure" on integration
2. Opens integration settings modal
3. User enters credentials/settings
4. Clicks "Test Connection"
5. System validates connection
6. User saves configuration

### 4. Manage Feature Flags
1. User toggles feature flag
2. Shows warning if experimental
3. Confirms change
4. Feature enabled/disabled immediately
5. Logs change in audit trail

### 5. Configure SSO
1. User clicks "Configure SAML"
2. Opens SAML configuration modal
3. Enters IdP metadata
4. Maps attributes
5. Clicks "Test SSO"
6. System tests connection
7. User saves configuration

### Settings Validation
```typescript
async function validateSettings(settings: SystemSettings) {
  const errors = [];

  // Validate URLs
  if (!isValidUrl(settings.general.applicationUrl)) {
    errors.push('Invalid application URL');
  }

  // Validate email
  if (!isValidEmail(settings.general.supportEmail)) {
    errors.push('Invalid support email');
  }

  // Validate password policy
  if (settings.security.passwordPolicy.minLength < 8) {
    errors.push('Minimum password length must be at least 8');
  }

  // Validate IP allowlist
  for (const ip of settings.security.ipAllowlist) {
    if (!isValidCIDR(ip)) {
      errors.push(`Invalid IP/CIDR: ${ip}`);
    }
  }

  return errors;
}
```
