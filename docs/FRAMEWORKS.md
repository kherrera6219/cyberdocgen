## Security & Information Governance

### ISO 27001:2022 - Information Security Management System
**93 Annex A Controls** across 4 categories (75 templates)

**Overview**: The international standard for information security management systems (ISMS). Version 2022 introduced significant changes including 11 new controls and consolidated structure.

**Four Control Categories**:
- **Organizational Controls** (37 controls) - Governance, policies, third-party management, access control
- **People Controls** (8 controls) - HR security, training, awareness
- **Physical Controls** (14 controls) - Facility protection, equipment security
- **Technological Controls** (34 controls) - Encryption, system monitoring, logging, malware defense

**New Controls in 2022**: Threat intelligence, Cloud security, ICT readiness for business continuity, Physical security monitoring, Configuration management, Information deletion, Data masking, Data leakage prevention, Monitoring activities, Web filtering, Secure coding

**Our Templates Include**:
- System Security Plan (SSP)
- Risk Assessment and Treatment Plan
- Statement of Applicability (SoA)
- Information Security Policy
- Access Control Policy
- Incident Response Plan
- Business Continuity Plan
- Supplier Security Management
- Complete Annex A control implementation guides

## Federal & Government Compliance

### FedRAMP (Federal Risk and Authorization Management Program)
**Three impact levels based on NIST 800-53 Rev 5**

#### FedRAMP Low Baseline
**156 controls** for low-impact federal systems (7 templates)

**When to Use**: Systems where loss of confidentiality, integrity, or availability would have limited adverse effect on operations, assets, or individuals.

**Our Templates Include**:
- System Security Plan (SSP) - Low Impact
- Security Assessment Plan (SAP)
- Security Assessment Report (SAR)
- Plan of Action & Milestones (POA&M)
- Continuous Monitoring Strategy
- Incident Response Plan
- Configuration Management Plan

#### FedRAMP Moderate Baseline
**323 controls** for moderate-impact systems (12 templates)

**When to Use**: Most common FedRAMP baseline. Systems where loss could result in serious harm to operations, finances, or individuals.

**Additional Templates Include**:
- Enhanced SSP with all 323 controls
- Privacy Threshold Analysis
- Privacy Impact Assessment
- E-Authentication Risk Assessment
- Supply Chain Risk Management Plan
- Cryptographic Key Management Plan

#### FedRAMP High Baseline
**413 controls** for high-impact federal systems (25 templates)

**When to Use**: Systems where loss could result in severe or catastrophic adverse effects on operations, assets, or individuals, including loss of life or serious life-threatening injuries.

**Comprehensive Coverage**:
- Complete NIST 800-53 high baseline implementation
- Enhanced security controls documentation
- Full audit and compliance package
- All 17 SSP appendices
- Continuous monitoring documentation

### NIST Cybersecurity Framework 2.0
**6 functions, 22 categories, 107 subcategories** (32 templates)

**Overview**: Released 2024, NIST CSF 2.0 provides a policy framework of computer security guidance for organizations to assess and improve their ability to prevent, detect, and respond to cyber attacks.

**Six Functions** (GOVERN is new in v2.0):
1. **GOVERN (GV)** - NEW: Cybersecurity risk governance and oversight
2. **IDENTIFY (ID)** - Asset management, business environment, governance, risk assessment
3. **PROTECT (PR)** - Access control, awareness training, data security, protective technology
4. **DETECT (DE)** - Anomalies and events, continuous monitoring, detection processes
5. **RESPOND (RS)** - Response planning, communications, analysis, mitigation, improvements
6. **RECOVER (RC)** - Recovery planning, improvements, communications

**Our Templates Include**:
- CSF 2.0 Implementation Roadmap
- Governance framework documentation (GOVERN function)
- Asset inventory and management
- Risk assessment methodology
- Access control procedures
- Security awareness training programs
- Incident detection and monitoring plans
- Incident response procedures
- Recovery and business continuity plans

### CCPA/CPRA (California Consumer Privacy Act)
**California privacy law with 2024 CPRA amendments** (30 templates)

**Overview**: California's comprehensive privacy law granting consumers rights over their personal information. CPRA (California Privacy Rights Act) amendments took effect January 1, 2023.

**Consumer Rights** (2024):
- Right to KNOW what personal information is collected
- Right to DELETE personal information
- Right to OPT-OUT of sale/sharing
- Right to CORRECT inaccurate information

**Enforcement**: California Privacy Protection Agency (CPPA) established by CPRA with penalties up to $7,500 per intentional violation.

**Our Templates Include**:
- CCPA/CPRA compliance implementation plan
- Consumer request handling procedures (DSAR)
- Privacy policy templates
- Do Not Sell disclosure
- Sensitive personal information handling
- Data inventory and mapping

### PCI DSS 4.0 (Payment Card Industry Data Security Standard)
**12 requirements, 64 new controls** (23 templates)

**Overview**: Security standard for organizations that handle credit cards. Version 4.0 effective March 31, 2024 (v3.2.1 retired).

**12 Requirements** across 6 goals:

**Build and Maintain a Secure Network**:
1. Install and maintain network security controls
2. Apply secure configurations

**Protect Cardholder Data**:
3. Protect stored account data
4. Protect cardholder data with strong cryptography during transmission

**Maintain a Vulnerability Management Program**:
5. Protect systems against malware
6. Develop and maintain secure systems

**Implement Strong Access Control Measures**:
7. Restrict access to cardholder data by business need-to-know
8. Identify users and authenticate access

**Regularly Monitor and Test Networks**:
9. Restrict physical access to cardholder data
10. Log and monitor all access to network resources and cardholder data

**Maintain an Information Security Policy**:
11. Test security systems and networks regularly
12. Support information security with organizational policies and programs

**Our Templates Include**:
- PCI DSS compliance program
- Network security architecture
- Cardholder data encryption procedures
- Access control policies
- Vulnerability management program
- Penetration testing procedures
- Security awareness training
- Incident response plan
- All 12 requirement implementation guides

### NIS2 (Network and Information Security Directive 2)
**EU cybersecurity requirements for 18 critical sectors** (20 templates)

**Overview**: EU directive establishing unified cybersecurity framework across 18 critical sectors. Effective October 17, 2024 (replaces NIS1).

**Key Requirements**:
- **Access Control**: Multi-factor authentication (MFA) mandatory
- **Encryption**: Data encrypted at rest and in transit
- **Network Security**: Firewalls and IDS/IPS
- **Incident Reporting**:
  - 24-hour early warning for significant incidents
  - 72-hour full incident report
  - Final report within one month

**Penalties**:
- Essential entities: up to €10 million or 2% worldwide annual turnover
- Important entities: up to €7 million or 1.4% worldwide annual turnover

**Our Templates Include**:
- NIS2 compliance program
- Cybersecurity risk management framework
- Multi-factor authentication implementation
- Data encryption standards
- Network security controls
- Incident notification procedures (24/72-hour)
- Supply chain security requirements
- Business continuity planning

## Cloud & Zero Trust Architecture

### AWS Well-Architected Framework
**6 pillars for cloud excellence** (23 templates)

**Overview**: AWS best practices for designing and operating reliable, secure, efficient, and cost-effective systems in the cloud. Sustainability pillar added in late 2021.

**Six Pillars**:

1. **Operational Excellence** - Running and monitoring systems, continually improving processes
2. **Security** - Protecting information and systems, data confidentiality and integrity
3. **Reliability** - Workload performing intended functions, recovery from failures
4. **Performance Efficiency** - Efficient resource allocation, monitoring performance
5. **Cost Optimization** - Avoiding unnecessary costs, right-sizing resources
6. **Sustainability** - Minimizing environmental impacts, maximizing resource utilization

**Our Templates Include**:
- Well-Architected Framework Review documentation
- Pillar-specific assessment questionnaires
- Security best practices implementation
- Reliability design patterns
- Performance optimization guides
- Cost optimization strategies
- Sustainability metrics and tracking
- Multi-pillar remediation plans

---

### CISA Zero Trust Maturity Model v2.0
**5 pillars, 3 cross-cutting capabilities** (23 templates)

**Overview**: CISA's framework for federal agencies transitioning to Zero Trust Architecture. Version 2.0 published April 2023 with FY2024 implementation objectives.

**Five Pillars**:
1. **Identity** - User and entity attributes and authentication
2. **Devices** - Asset inventory, configuration management, endpoint security
3. **Networks** - Microsegmentation, encrypted communications, network visibility
4. **Applications and Workloads** - Application security, API security, workload isolation
5. **Data** - Data classification, encryption, DLP, access controls

**Three Cross-Cutting Capabilities**:
- Visibility and Analytics
- Automation and Orchestration
- Governance

**Maturity Stages**: Traditional → Initial → Advanced → Optimal

**Our Templates Include**:
- Zero Trust Architecture implementation roadmap
- Maturity assessment by pillar
- Identity and access management (IAM) procedures
- Device inventory and trust assessment
- Network segmentation plans
- Microsegmentation architecture
- Application security controls
- Data classification and protection
- Visibility and analytics framework
- Automation playbooks
- Governance and policy documentation
