## General Help

Open a GitHub issue when you need help with:

- Local setup or environment configuration
- Build, packaging, or deployment failures
- Runtime bugs or regressions
- Documentation mismatches
- Feature requests or implementation gaps

When filing an issue, include:

- Deployment mode: `cloud` or `local`
- OS and runtime details
- The exact command or page path involved
- Logs, stack traces, or screenshots when available
- Steps to reproduce

## Documentation Corrections

If the code and docs disagree, open an issue or submit a pull request with:

- The incorrect document path
- The code path that is the actual source of truth
- The concrete mismatch

## Security Issues

Do not file public issues for security vulnerabilities.

Use [SECURITY.md](SECURITY.md) for private disclosure guidance.

## Private Or Customer-Specific Support

If you are operating a private deployment, customer environment, or release-signing workflow, use the existing maintainer or customer channel already associated with your deployment instead of filing a public issue with sensitive details.
