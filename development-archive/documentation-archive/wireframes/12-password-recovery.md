## Interactions

### 1. Request Reset Flow
1. User enters email address
2. Clicks "Send Reset Link"
3. System validates email format
4. System checks if account exists
5. Sends reset email (if account found)
6. Shows confirmation screen (always, for security)

### 2. Reset Password Flow
1. User clicks link in email
2. System validates token
3. If valid, shows reset form
4. User enters new password
5. System validates password strength
6. User confirms password
7. System updates password
8. Shows success message
9. Redirects to login

### 3. Resend Email
1. User clicks "Resend Email"
2. System checks rate limit (1 email/minute)
3. Sends new email
4. Shows confirmation

---

## Security Considerations

### Token Security
- Reset tokens expire after 1 hour
- Tokens are single-use only
- Tokens invalidated after password change
- Secure random token generation

### Rate Limiting
- Max 3 reset requests per email per hour
- Max 5 reset attempts per IP per hour
- Progressive delays on failed attempts

### Email Security
- Always show confirmation (don't reveal if email exists)
- Include IP address and timestamp in email
- Include device/browser information
- Link to "Wasn't me?" support

## Technical Notes

### API Endpoints
```
POST /api/auth/forgot-password
  Body: { email: string }
  Response: { message: "Reset email sent" }

POST /api/auth/reset-password
  Body: { token: string, newPassword: string }
  Response: { message: "Password reset successful" }

POST /api/auth/verify-reset-token
  Body: { token: string }
  Response: { valid: boolean, email: string }
```

### Validation Rules
```typescript
// Email validation
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/

// Password requirements
- Minimum 12 characters
- At least one uppercase letter
- At least one lowercase letter
- At least one number
- At least one special character (!@#$%^&*)
- Not in common password list
- Not same as previous password
```
